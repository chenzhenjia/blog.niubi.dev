使用kubespray部署kubernetes集群

发表于 2017-11-28 更新于 2017-12-14

准备工作

  1. 因为是部署集群，所以需要两台机器，(我使用的是digitalocean这个云提供商)
  2. 内存至少1.5GB 以上，不然会安装失败。
  3. 需要每一台机器都能 ping 通

我的一些配置

  1. 系统为 centos7
  2. 节点的信息
    这里使用digitalocean提供的内网地址
    ip 地址 角色 内存 CPU
    10.132.37.154 master 3G 1
    10.132.45.167 node1 3G 1

准备安装

  1. 修改主节点的 hosts 文件，把节点ip 和名字写进去
    vi /etc/hosts

  2. 所有节点无密码登录

    1. 生成无密码的密钥对，用来进行主节点到副节点之间的无密码登录。
      直接全都回车，生成的密钥对在 ~/.ssh 目录下面，默认名称为 id_rsa和id_rsa.pub
    2. 把公钥分别写到主节点和副节点的~/.ssh/authorized_keys 文件中。

    注意：每个节点都有添加这个公钥，不管是本机还是其他机器

  3. 在主节点安装pip

    1. centos 安装
      easy_install pip
    2. ubuntu 安装

  4. centos 需要安装 git

  5. 使用 pip 安装 ansible 和 kubespray

  6. 修改所有节点的 /etc/resolv.conf

  7. 添加 kubespray 的配置文件

  8. 用 kubespray 生成节点的配置文件
    会在 ~/ 目录下生成一个 .kubespray 文件夹

  9. 编辑 inventory.cfg 文件

  10. 在 ~/.kubespray 目录下执行部署 k8s 命令。
    注意: 在 digitalocean 的 centos7 上会遇到一个 Check_certs | Set 'sync_certs' to true 的错误。解决方法
    显示了 Kubernetes deployed successfuly 则部署成功

  11. 验证 k8s 集群

增加节点

  1. 把新增的节点 ip 配置到 hosts 文件中
    vi /etc/hosts
  2. 在 ~/.kubespray 目录执行命令
    cd ~/.kubespray
    ansible-playbook -i inventory/inventory.cfg scale.yml -b -v --private-key=~/.ssh/id_rsa -l node2
